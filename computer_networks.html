OVERVIEW & MODELS

OSI 7-Layer Model

The Open Systems Interconnection (OSI) model standardises network communication into 7 abstraction layers. Each layer serves the layer above it and is served by the layer below.

7ApplicationDataHTTP, HTTPS, FTP, SMTP, DNS, SNMP, Telnet
6PresentationDataSSL/TLS, JPEG, MPEG, ASCII, Encryption/Compression
5SessionDataNetBIOS, RPC, PPTP, SIP
4TransportSegmentTCP, UDP, SCTP — Port numbers live here
3NetworkPacketIP (v4/v6), ICMP, OSPF, BGP, RIP, ARP*
2Data LinkFrameEthernet, PPP, 802.11 MAC, VLAN, STP
1PhysicalBitsEthernet physical, USB, Bluetooth, DSL, fiber
Mnemonic (top→bottom): "All People Seem To Need Data Processing" | (bottom→top): "Please Do Not Throw Sausage Pizza Away"

TCP/IP Model (4-Layer)

Key Differences: OSI vs TCP/IP

OSI 7 Layers

Conceptual/reference model. Strict separation. Rarely implemented purely. Good for troubleshooting. Has Presentation and Session layers.

TCP/IP 4 Layers

Practical, deployed model. Combines OSI layers 5-7 into Application. No strict separation. The internet runs on this.

Encapsulation & Decapsulation

As data travels down the stack, each layer adds a header (and sometimes a trailer). This is encapsulation. At the receiver, each layer strips its header — decapsulation. The PDU name changes at each layer: Data → Segment → Packet → Frame → Bits.

App Layer:     [  DATA  ]
Transport:     [ TCP HDR | DATA ]
Network:       [ IP HDR | TCP HDR | DATA ]
Data Link:     [ ETH HDR | IP HDR | TCP HDR | DATA | ETH TRAILER ]
Physical:      101010010110... (bits on wire)

PHYSICAL LAYER

Layer 1 deals with raw bit transmission over a physical medium. It defines electrical voltages, frequencies, pin layouts, cable specifications, and signal encoding.

Nyquist & Shannon Theorems

Nyquist: Max Bit Rate = 2 × B × log₂(V)   |   B = bandwidth (Hz), V = signal levels
Shannon: Max Capacity = B × log₂(1 + S/N)   |   S/N = signal-to-noise ratio

Nyquist applies to noiseless channels; Shannon gives the theoretical maximum for noisy channels regardless of encoding.

Multiplexing Techniques

FDM Frequency Division

Different signals on different frequency bands simultaneously. Used in cable TV, analog radio.

TDM Time Division

Each signal gets a time slot in a repeating frame. Used in T1/E1, SONET/SDH.

WDM Wavelength Division

Different wavelengths (colors) of light on a single fiber. DWDM supports 80+ channels.

CDM Code Division

Signals spread over entire bandwidth using orthogonal codes. Used in 3G CDMA, GPS.

OFDM Orthogonal FDM

Many closely spaced subcarriers; orthogonal so no inter-carrier interference. 4G/5G, 802.11a/g/n/ac/ax.

STDM Statistical TDM

Slots allocated dynamically based on demand. More efficient than fixed TDM.

Transmission Modes

  • Simplex — One direction only (e.g., TV broadcast, keyboard)
  • Half-Duplex — Both directions but not simultaneously (e.g., walkie-talkie, old hubs)
  • Full-Duplex — Simultaneous both directions (e.g., telephone, modern switches)

DATA LINK LAYER

Layer 2 provides node-to-node delivery, framing, MAC addressing, error detection, and access control over a single link.

MAC Addresses

A 48-bit (6-byte) hardware address burned into a NIC, written as AA:BB:CC:DD:EE:FF. First 3 bytes = OUI (vendor), last 3 = device. Broadcast MAC = FF:FF:FF:FF:FF:FF.

Ethernet Frame Structure

| Preamble (7B) | SFD (1B) | Dst MAC (6B) | Src MAC (6B) | EtherType (2B) | Payload (46–1500B) | FCS (4B) |
  10101010...    10101011                                   0x0800=IPv4
                                                            0x86DD=IPv6
                                                            0x0806=ARP

MTU (Maximum Transmission Unit) for Ethernet = 1500 bytes. Jumbo frames extend to 9000 bytes in data centers.

MAC Sub-layers

LLC — Logical Link Control (802.2)

Handles multiplexing, error checking, flow control between MAC and network layer.

MAC — Media Access Control

Controls how devices access the shared medium. Handles CSMA/CD, CSMA/CA.

ARP (Address Resolution Protocol)

Maps an IP address → MAC address on a LAN. Works at Layer 2/3 boundary.

Host A wants to reach 192.168.1.5 but doesn't know its MAC.
1. A broadcasts ARP Request: "Who has 192.168.1.5? Tell 192.168.1.1"
2. Host with IP 192.168.1.5 responds unicast with its MAC.
3. A caches IP→MAC in ARP table (default 20 min TTL).

Gratuitous ARP: Host ARPs for its own IP.
  Used for: IP conflict detection, updating ARP caches after failover.
ARP Poisoning: Attacker sends fake ARP replies → MiTM. Defense: Dynamic ARP Inspection (DAI).

VLANs & 802.1Q Tagging

VLANs logically segment a physical network. A 802.1Q tag (4 bytes) is inserted into the Ethernet frame after the source MAC:

| Dst MAC | Src MAC | [802.1Q: TPID=0x8100 | PCP(3b) | DEI(1b) | VID(12b)] | EtherType | Payload | FCS |
                                                                     ↑ VLAN ID 1–4094
  • Access port — carries traffic for one VLAN; untagged to end device
  • Trunk port — carries multiple VLANs with 802.1Q tags between switches
  • Native VLAN — untagged traffic on a trunk belongs to native VLAN (default VLAN 1)
  • Inter-VLAN routing — requires Layer 3 (router or Layer-3 switch). "Router-on-a-stick" uses sub-interfaces.

CIDR (Classless Inter-Domain Routing) replaced classful addressing. Subnet mask specifies how many bits are network bits.

# of usable hosts = 2^(host bits) − 2   |   # of subnets = 2^(borrowed bits)

VLSM (Variable Length Subnet Masking)

Allows different-sized subnets within the same IP space. Critical for efficient IP allocation. Always subnet the largest requirement first.

Supernetting / Route Summarization

Combining multiple contiguous subnets into a single summary route. Reduces routing table size. Requires contiguous block aligned on power-of-2 boundary.

Subnetting trick: The subnet address = IP AND mask. Broadcast = subnet OR (NOT mask). First host = subnet + 1. Last host = broadcast − 1.

IPv6

128-bit addresses. Written as 8 groups of 4 hex digits: 2001:0db8:85a3:0000:0000:8a2e:0370:7334

Abbreviation rules: (1) leading zeros in each group can be omitted; (2) one sequence of consecutive all-zero groups can be replaced with ::

Address Types

TypePrefixScopeDescription
Loopback::1/128—Equivalent to IPv4 127.0.0.1
Link-Localfe80::/10LinkAuto-configured, not routable. Always on every interface.
Unique Localfc00::/7SiteRFC 4193. Like private RFC 1918.
Global Unicast2000::/3GlobalPublicly routable. Currently 2000:: – 3FFF::
Multicastff00::/8Variousff02::1 = all nodes; ff02::2 = all routers
AnycastSame as unicastGlobalAssigned to multiple interfaces; routed to nearest

IPv6 Header Improvements over IPv4

  • Fixed 40-byte header (no options field bloat)
  • No fragmentation by routers — only source does it using extension headers
  • No checksum — leaves verification to upper layers (TCP/UDP)
  • Flow Label field for QoS without deep packet inspection
  • Extension headers replace IPv4 options (Hop-by-Hop, Routing, Fragment, Authentication, ESP)

NDP (Neighbor Discovery Protocol)

Replaces ARP in IPv6. Uses ICMPv6 messages:

  • RS/RA — Router Solicitation / Router Advertisement (stateless autoconfiguration)
  • NS/NA — Neighbor Solicitation / Neighbor Advertisement (ARP replacement)
  • Redirect — Router tells host of better next hop

SLAAC (Stateless Address Autoconfiguration)

Host generates its own IPv6 address using RA prefix + EUI-64 (from MAC address) or random bits (RFC 7217). No DHCP server needed.

EUI-64: MAC 00:1A:2B:3C:4D:5E → 021A:2BFF:FE3C:4D5E (insert FFFE, flip 7th bit)

How PAT Works

Private         Router (PAT)      Internet
10.0.0.2:54321 → 203.0.113.1:1024 → Server
10.0.0.3:54321 → 203.0.113.1:1025 → Server

NAT Table: [203.0.113.1:1024 ↔ 10.0.0.2:54321]
           [203.0.113.1:1025 ↔ 10.0.0.3:54321]
NAT breaks end-to-end connectivity — problematic for VoIP, gaming, P2P. Solutions: STUN, TURN, ICE (used in WebRTC), UPnP, NAT-PMP.

NAT Traversal Techniques

  • STUN — Client discovers its public IP:port by querying a STUN server. Works for "cone" NATs.
  • TURN — Traffic relayed through a server. Works for symmetric NAT but costly.
  • ICE — Framework combining STUN + TURN for WebRTC peer connections.
  • Hole Punching — Both peers simultaneously send packets to each other's public endpoints, punching holes in NAT tables.

DHCP

Dynamic Host Configuration Protocol (UDP 67 server, 68 client) dynamically assigns IP addresses and network config.

DORA Process

Client           Broadcast                DHCP Server
  │──── DISCOVER ────────────────────────────▶│  (src: 0.0.0.0:68, dst: 255.255.255.255:67)
  │◀─── OFFER ─────────────────────────────── │  (offers IP + lease time + gateway + DNS)
  │──── REQUEST ──────────────────────────────▶│  (client accepts the offer, broadcasts to all servers)
  │◀─── ACK ───────────────────────────────── │  (server confirms; client configures IP)

DHCP Lease & Options

  • Option 1 — Subnet mask
  • Option 3 — Default gateway (router)
  • Option 6 — DNS server
  • Option 15 — Domain name
  • Option 43 — Vendor-specific info (e.g., VoIP phone config)
  • Option 82 — DHCP relay agent info (circuit ID, remote ID)
  • At 50% of lease: client tries to renew with unicast to server
  • At 87.5% of lease: client tries to rebind with broadcast to any server

DHCPv6 vs SLAAC

DHCPv6 (Stateful)

Server assigns and tracks IPv6 addresses. RA has M-flag set. Server knows which IP is assigned to which host.

SLAAC (Stateless)

Host auto-configures from RA prefix. No server needed. But DNS still needs to be configured (via DHCPv6 or RA RDNSS option).

DNSSEC

DNS Security Extensions adds digital signatures to DNS responses. Chain of trust from root → TLD → authoritative. Prevents DNS cache poisoning (Kaminsky attack). Does NOT provide confidentiality — use DoT or DoH for that.

DoT & DoH

DoT DNS over TLS — Port 853

Encrypts DNS traffic over dedicated TLS connection. Distinguishable from other traffic (port 853). Easy for network admins to monitor.

DoH DNS over HTTPS — Port 443

DNS embedded in HTTPS traffic. Indistinguishable from web browsing. Harder to block. Used by browsers (Firefox, Chrome by default).

DNS Cache Poisoning (Kaminsky Attack)

Attacker floods resolver with fake DNS responses with random transaction IDs hoping to win the race before the real response. Fixed by: source port randomization + DNSSEC. Bailiwick checking also helps.

Client                          Server
  │──── FIN ──────────────────────▶│  (Client done sending)
  │◀─── ACK ───────────────────────│  (Server ACKs FIN)
  │◀─── FIN ───────────────────────│  (Server done sending)

TCP State Machine

CLOSED → LISTEN (server) or SYN_SENT (client)
SYN_SENT → SYN_RCVD (server) → ESTABLISHED (both after handshake)
ESTABLISHED → FIN_WAIT_1 (initiator) → FIN_WAIT_2 → TIME_WAIT → CLOSED
ESTABLISHED → CLOSE_WAIT (receiver) → LAST_ACK → CLOSED

Sliding Window & Flow Control

The receive window (rwnd) advertised in the ACK tells sender how much buffer space is available. Sender can only have min(cwnd, rwnd) bytes unacknowledged in flight.

Selective Acknowledgment (SACK)

TCP option that allows receiver to ACK non-contiguous blocks, so sender only retransmits genuinely lost segments (not everything after the gap). Negotiated in SYN/SYN-ACK.

TCP Timestamps Option

Two values: TSval (sender's clock) and TSecr (echo of peer's TSval). Enables accurate RTT measurement and PAWS (Protection Against Wrapped Sequence numbers) — prevents old duplicate packets from older connections being accepted.

TCP Fast Retransmit & Fast Recovery

  • Fast Retransmit: After 3 duplicate ACKs, retransmit without waiting for retransmit timeout (RTO)
  • Fast Recovery: After fast retransmit, set ssthresh = cwnd/2, cwnd = ssthresh + 3. Continue without going to slow start.

Nagle's Algorithm

Reduces small packet proliferation: if there's unACKed data in flight and new data is smaller than MSS, buffer it until ACK arrives. Can cause latency in interactive apps. Disabled with TCP_NODELAY socket option (used in SSH, games, trading).

TCP SYN Cookies

Defense against SYN flood (DDoS). Server doesn't allocate resources on SYN; instead encodes state in the ISN (sequence number). Only allocates resources on receiving valid ACK. Drawback: TCP options (SACK, window scale) can't be remembered during handshake.

CONGESTION CONTROL

Core Variables

  • cwnd (congestion window) — sender-side limit on unACKed bytes
  • ssthresh (slow start threshold) — where slow start transitions to AIMD
  • rwnd (receive window) — receiver-side buffer limit
  • Effective window = min(cwnd, rwnd)

TCP Reno (Classic)

Phase 1: Slow Start
  cwnd starts at 1 MSS. cwnd doubles every RTT (exponential) until cwnd ≥ ssthresh.

Phase 2: Congestion Avoidance (AIMD)
  cwnd increases by 1 MSS per RTT (linear). Additive Increase.

On 3 dup ACKs (packet loss assumed):
  ssthresh = cwnd / 2
  cwnd = ssthresh + 3  (fast recovery)
  retransmit lost segment

On timeout (severe congestion):
  ssthresh = cwnd / 2
  cwnd = 1 MSS         (back to slow start)  — Multiplicative Decrease

TCP Cubic (Default in Linux)

Uses a cubic function of time since last congestion event to set cwnd. More aggressive at high BDP (bandwidth-delay product) than Reno. W(t) = C(t − K)³ + Wmax. Default in Linux since 2.6.19.

TCP BBR (Bottleneck Bandwidth and RTT)

Google's model-based congestion control (2016). Instead of reacting to loss, BBR continuously estimates bottleneck bandwidth and minimum RTT, then sends at exactly that rate. Significantly better throughput on lossy/high-latency links (e.g., ocean cables, WiFi). Used in YouTube, Google CDN.

QUIC Congestion Control

QUIC (used in HTTP/3) implements its own congestion control at the application layer (typically Cubic or BBR). Avoids TCP head-of-line blocking since each QUIC stream is independent.

ECN (Explicit Congestion Notification)

Routers mark packets (ECT bit in IP header) instead of dropping them when buffers fill. Receiver echoes congestion signal back. Sender reduces rate without dropping. Requires support from both endpoints and all routers.

ROUTING PROTOCOLS

Routing Protocol Categories

IGP — Interior Gateway Protocol

Within a single AS (Autonomous System). Examples: RIP, OSPF, EIGRP, IS-IS.

EGP — Exterior Gateway Protocol

Between ASes. BGP is the only EGP in use today. Powers the internet.

Distance Vector

Each router knows distance/direction to every destination, not full topology. Periodic updates. RIP, EIGRP (hybrid).

Link State

Each router knows the full topology. Runs Dijkstra SPF. Faster convergence. OSPF, IS-IS.

RIP (Routing Information Protocol)

  • Distance Vector, metric = hop count (max 15; 16 = unreachable)
  • RIPv1: classful, broadcast. RIPv2: classless, multicast 224.0.0.9. RIPng: IPv6.
  • Updates every 30 seconds. Convergence slow. Split horizon + poison reverse prevent loops.
  • Counting to infinity problem — mitigated by split horizon and triggered updates.

OSPF (Open Shortest Path First)

Link-state protocol. Uses Dijkstra's SPF algorithm. Administrative distance = 110.

OSPF Packet Types: Hello | DBD (Database Description) | LSR | LSU | LSAck
OSPF States: Down → Init → 2-Way → Exstart → Exchange → Loading → Full
DR/BDR election on multi-access networks (highest priority, then highest Router ID)
Hello interval: 10s (point-to-point / broadcast), 30s (NBMA)
Dead interval: 4× Hello interval
  • Area 0 (Backbone) — all other areas must connect to Area 0
  • ABR — Area Border Router (connects areas to backbone)
  • ASBR — AS Boundary Router (connects OSPF to other routing domains)
  • LSA Types: Type 1 (Router), Type 2 (Network), Type 3 (Summary), Type 4 (ASBR Summary), Type 5 (External), Type 7 (NSSA External)
  • Stub areas block Type 5 LSAs; Totally Stubby block Type 3+4+5; NSSA allow external routes as Type 7

BGP (Border Gateway Protocol)

The Internet's routing protocol. Path-vector, uses TCP port 179. AD = 20 (eBGP), 200 (iBGP).

BGP Message Types: OPEN | UPDATE | NOTIFICATION | KEEPALIVE
BGP States: Idle → Connect → Active → OpenSent → OpenConfirm → Established
eBGP: between different ASes (TTL=1 by default)
iBGP: within same AS (requires full mesh or route reflectors/confederations)

BGP Path Selection (in order)

1. Highest Weight (Cisco proprietary, local)
2. Highest Local Preference (within AS)
3. Locally originated routes (network/redistribute/aggregate)
4. Shortest AS_PATH
5. Lowest Origin (IGP < EGP < incomplete)
6. Lowest MED (from neighboring AS)
7. eBGP over iBGP
8. Lowest IGP metric to BGP next-hop
9. Oldest eBGP route
10. Lowest Router ID
11. Lowest Neighbor IP
BGP Hijacking: Attacker announces more-specific prefix for victim's IP space, attracting traffic. Mitigated by RPKI (Resource Public Key Infrastructure) which cryptographically validates route origin.

EIGRP (Enhanced IGRP)

Cisco proprietary hybrid (distance-vector + some link-state features). Uses DUAL algorithm. Metric based on bandwidth + delay (+ load, reliability optionally). Supports VLSM, fast convergence. Protocol number 88.

IS-IS (Intermediate System to Intermediate System)

Link-state, ISO protocol (not IP-based itself). Used heavily by ISPs. Like OSPF but runs directly on Data Link layer. L1 = within area, L2 = between areas. Preferred over OSPF by large carriers due to scalability and flexibility.

L3 Switch2+3ASIC hardware routingWire-speed inter-VLAN routing

MAC Address Table & CAM Table

Switches learn source MAC addresses and associate them with ports. On unknown destination MAC: flood all ports. CAM table entries age out (default 300s). CAM table overflow attack: flood with fake MACs → table fills → switch acts like hub → sniff traffic. Mitigated by port security (limit MACs per port).

Switch Forwarding Modes

  • Store-and-Forward — receive entire frame, check FCS, then forward. Most reliable. Higher latency.
  • Cut-Through — forward after reading just destination MAC (14 bytes). Lowest latency. Propagates corrupt frames.
  • Fragment-Free — forward after reading 64 bytes (minimum Ethernet frame). Balance between above two.
15

STP & REDUNDANCY

Spanning Tree Protocol (802.1D)

Prevents Layer 2 loops by blocking redundant paths. Creates a loop-free tree.

1. Elect Root Bridge (lowest Bridge ID = Priority + MAC)
2. Each non-root switch elects Root Port (lowest cost to root)
3. Each segment elects Designated Port (lowest cost to root)
4. All other ports → Blocking

Port States: Blocking(20s) → Listening(15s) → Learning(15s) → Forwarding
Total convergence: ~50 seconds (problematic!)

RSTP (Rapid STP, 802.1w)

Faster convergence (~1-2s). New port roles: Alternate (backup for root port), Backup (backup for designated). Proposal/Agreement mechanism replaces listening/learning delays. Backward compatible with 802.1D.

MSTP (Multiple STP, 802.1s)

Maps multiple VLANs to multiple STP instances. Allows load balancing across redundant links per VLAN.

PVST+ & Rapid PVST+

Cisco proprietary: separate STP instance per VLAN. Allows different root bridges per VLAN for load balancing.

Etherchannel / LAG / LACP

Bundles multiple physical links into a single logical link. STP sees it as one port (no blocking). LACP (802.3ad) is the standard negotiation protocol. PAgP is Cisco proprietary.

SESSION & PRESENTATION LAYER

Session Layer (L5)

Manages sessions: establishment, maintenance, synchronisation, and termination. Provides checkpoints (sync points) so large transfers can resume after failure. Protocols: NetBIOS, RPC, NFS, SMB (partially), SIP.

Presentation Layer (L6)

Handles data representation, encryption, compression, and character encoding. Ensures data sent by one system is readable by another. Examples: SSL/TLS encryption happens here, JPEG/MPEG compression, ASCII/Unicode translation, XDR (External Data Representation for RPC).

In practice, the TCP/IP model merges L5, L6, L7 into the Application layer. Modern frameworks (TLS, HTTP/2) handle these concerns within the application protocol.

HTTP/2

  • Binary framing — replaces text. More efficient to parse.
  • Multiplexing — multiple requests over single TCP connection simultaneously (no HOL blocking at HTTP layer)
  • Stream prioritization — clients can weight and depend streams
  • HPACK header compression — static + dynamic table; Huffman encoding
  • Server Push — server proactively sends resources client will need
  • TCP HOL blocking still exists — if a TCP segment is lost, all streams stall

HTTP/3 (QUIC-based)

  • Runs over QUIC (UDP-based transport protocol from Google)
  • 0-RTT connection setup — subsequent connections resume with 0 round trips
  • Independent streams — loss in one stream doesn't block others (TCP HOL blocking eliminated)
  • Connection migration — connection ID instead of 4-tuple; survives IP change (WiFi → cellular)
  • Mandatory TLS 1.3 — security built into QUIC itself
  • Uses UDP port 443

After upgrade, both sides can send frames anytime. Frame types: text, binary, ping, pong, close. Used in: live chat, real-time dashboards, multiplayer games, trading.

gRPC

Google's RPC framework. Uses HTTP/2 for transport and Protocol Buffers (protobuf) for serialization. More efficient than REST+JSON: binary format, schema-driven, streaming support.

// .proto definition

  rpc SayHello (HelloRequest) returns (HelloReply);
  rpc SayHelloStream (HelloRequest) returns (stream HelloReply); // server streaming

4 communication patterns:
  1. Unary (req → resp)
  2. Server streaming (req → stream of resp)
  3. Client streaming (stream of req → resp)
  4. Bidirectional streaming (stream ↔ stream)

SSE (Server-Sent Events)

One-directional: server → client over HTTP. Client opens a GET request; server streams text/event-stream. Simpler than WebSockets for server-push use cases. Auto-reconnect built-in. Used by Claude, ChatGPT for token streaming.

QoS — QUALITY OF SERVICE

Mechanisms to prioritize certain traffic types, ensure bandwidth guarantees, and reduce latency/jitter for sensitive applications (VoIP, video conferencing).

QoS Models

Best Effort

No guarantees. All traffic treated equally. Default internet model.

IntServ (RSVP)

Per-flow reservations. RSVP signals bandwidth needs. Doesn't scale to large networks.

DiffServ

Traffic classes using DSCP bits in IP header. Scalable. Per-hop behaviour (PHB).

DSCP (Differentiated Services Code Point)

6-bit field in IP header (in DSCP/ECN byte). Per-Hop Behaviors:

  • EF (Expedited Forwarding) — DSCP 46: Low latency/loss/jitter. VoIP, real-time video.
  • AF (Assured Forwarding) — AFxy: 4 classes (AF1x–AF4x) × 3 drop precedences. Business apps.
  • CS (Class Selector): CS0 (best effort) to CS7 (network control)
  • Default (DSCP 0): Best effort

Queuing Mechanisms

AlgorithmHow It WorksBest For
FIFOFirst in, first out. Simple.Low-traffic links
PQ (Priority Queuing)High-priority always served firstStrict latency needs; can starve lower priority
WFQ (Weighted Fair Queuing)Flows weighted by IP precedence; fair shareDefault on most Cisco interfaces < 2Mbps
CBWFQUser-defined classes with bandwidth guaranteesEnterprise QoS
LLQCBWFQ + strict priority queue for real-time trafficVoIP + data on same link

Traffic Shaping vs Policing

Traffic Shaping

Buffers excess traffic to conform to rate. Smooths bursts. Increases latency. Used outbound.

Traffic Policing

Drops or re-marks excess traffic. No buffering. Immediate enforcement. Used inbound/outbound.

SDN & NFV

Software-Defined Networking

Decouples the control plane (routing decisions) from the data plane (packet forwarding). A centralised SDN controller has a global view of the network and programs forwarding rules via southbound APIs (OpenFlow, NETCONF, gRPC).

┌─────────────────────────────────┐
│  Applications (North Bound API) │  ← REST, Intent-based APIs
├─────────────────────────────────┤
│     SDN Controller (Control)    │  ← OpenDaylight, ONOS, Ryu
├─────────────────────────────────┤
│  Data Plane (Switches/Routers)  │  ← OpenFlow, P4 switches
└─────────────────────────────────┘

OpenFlow

First SDN southbound protocol. Controller installs flow table entries in switches. Each entry = match fields (IP, port, MAC) + actions (forward, drop, modify, send to controller).

P4 (Programming Protocol-Independent Packet Processors)

Language to define custom packet processing pipelines in the data plane itself. More flexible than OpenFlow. Used in SmartNICs, programmable ASICs, hyperscaler networks.

NFV (Network Function Virtualization)

Runs network functions (firewalls, load balancers, IDS, NAT, WAN optimization) as software on commodity x86 servers instead of dedicated hardware. Managed by NFV orchestrator (MANO). Key standards: ETSI NFV-MANO, OpenStack, Kubernetes.

VXLAN (Virtual Extensible LAN)

Overlay network protocol encapsulating Layer 2 Ethernet frames in UDP (port 4789). Extends VLANs across Layer 3 boundaries. 24-bit VNI (VXLAN Network Identifier) → 16M segments vs VLAN's 4094. Used in VMware NSX, OpenStack Neutron, AWS VPC.

Outer Ethernet | Outer IP | Outer UDP (4789) | VXLAN Header (VNI 24b) | Inner Ethernet Frame

MPLS

Multiprotocol Label Switching — packets forwarded based on short fixed-length labels rather than IP lookups. Faster, supports TE (Traffic Engineering) and VPNs.

MPLS Label Stack

| Ethernet | MPLS Label (32b) | IP Header | Payload |

Label (20b) | EXP/TC (3b) | S (1b, bottom of stack) | TTL (8b)

MPLS VPNs (L3VPN)

Customer sites connected as if on private network. PE routers use VRFs (Virtual Routing and Forwarding tables) to separate customer routes. Uses MP-BGP to distribute VPN routes with Route Distinguishers (RD) and Route Targets (RT).

Segment Routing (SR-MPLS / SRv6)

Modern evolution of MPLS TE. Source encodes entire path as stack of segment IDs (SIDs). No per-flow state in network. SR-MPLS uses MPLS label stack; SRv6 uses IPv6 extension headers. Simplifies MPLS TE and enables Intent-Based Networking.

WiFi Key Attack Vectors

  • KRACK (Key Reinstallation Attack) — replays handshake messages to force nonce reuse in WPA2 CCMP. Patched by most vendors.
  • Dragonblood — side-channel attacks on WPA3 SAE handshake. Mostly patched.
  • Evil Twin / Deauth attack — rogue AP + 802.11 deauth frames (Management Frame Protection / 802.11w mitigates).
  • PMKID attack — offline dictionary attack against WPA2 without completing full 4-way handshake.

OFDMA in WiFi 6

WiFi 6 introduces OFDMA which divides a channel into Resource Units (RUs), allowing the AP to schedule transmissions from multiple clients simultaneously in the same channel — unlike WiFi 5 which serves one client per transmission. Greatly improves dense deployment performance.

Zero Trust Networking

"Never trust, always verify." Principles: micro-segmentation, least privilege, continuous authentication, assume breach. Implemented via identity-aware proxies (BeyondCorp, Zscaler), service mesh (Istio with mTLS), SASE (Secure Access Service Edge).

IPSec

Suite of protocols for Layer 3 encryption and authentication. Two modes:

  • Transport mode — encrypts/authenticates only payload; IP header unchanged. Used host-to-host.
  • Tunnel mode — entire IP packet encrypted; new IP header added. Used in VPNs (site-to-site).
  • AH (Authentication Header) — integrity + anti-replay. No encryption. Protocol 51.
  • ESP (Encapsulating Security Payload) — encryption + integrity + anti-replay. Protocol 50.
  • IKE (Internet Key Exchange) — IPSec key negotiation. IKEv1 uses UDP 500. IKEv2 is simpler and more secure.

TLS 1.3 Improvements

  • 1-RTT handshake (vs 2-RTT in TLS 1.2)
  • 0-RTT resumption — client sends application data immediately on reconnect (with replay risk)
  • Removed weak algorithms: RSA key exchange, MD5, SHA-1, RC4, DES, 3DES, CBC mode ciphers
  • Forward Secrecy mandatory — only (EC)DHE key exchange allowed
  • Encrypted handshake — certificates encrypted; SNI and ALPN visible but certificate hidden
  • Cipher suites: TLS_AES_128_GCM_SHA256, TLS_AES_256_GCM_SHA384, TLS_CHACHA20_POLY1305_SHA256

Certificate Chain & PKI

Root CA (self-signed, stored in OS/browser trust store)
  └── Intermediate CA (signed by Root)
        └── End-Entity Certificate (signed by Intermediate)
              Contains: Subject, Issuer, SANs, Public Key, Validity, Extensions, Signature

Certificate Transparency (CT)

All publicly-trusted certificates must be logged in public append-only CT logs. Browsers check for SCTs (Signed Certificate Timestamps). Prevents misissued/malicious certificates from going undetected.

HSTS (HTTP Strict Transport Security)

Server sends Strict-Transport-Security: max-age=31536000; includeSubDomains; preload. Browser refuses to connect over HTTP for the specified duration. Preload lists baked into browsers.

SNI (Server Name Indication)

TLS extension that lets client specify hostname during handshake, before certificate is sent. Enables multiple TLS certificates on same IP. Visible in plaintext in TLS 1.2/1.3 (ECH — Encrypted Client Hello aims to fix this).

OCSP & CRL

  • CRL (Certificate Revocation List) — CA publishes list of revoked serial numbers. Can be large; periodic updates.
  • OCSP (Online Certificate Status Protocol) — real-time query for single cert status. Privacy risk (CA sees who connects where).
  • OCSP Stapling — server fetches and caches OCSP response, staples it to TLS handshake. No privacy/latency issue.

ERROR DETECTION & CORRECTION

Parity Bits

Single parity bit: detects 1-bit errors, cannot correct. Even parity: total 1s even. Odd parity: total 1s odd. Two-dimensional parity can detect and correct single-bit errors.

CRC (Cyclic Redundancy Check)

Polynomial division of data treated as binary polynomial. Appended CRC remainder allows receiver to detect burst errors. CRC-32 (Ethernet, ZIP, PNG): detects all burst errors ≤ 32 bits. CRC-16: serial protocols.

Checksum (IP/TCP/UDP)

One's complement sum of 16-bit words. Simple and fast. Weaker than CRC (doesn't detect certain multi-bit errors). IP header checksum recalculated at each hop (TTL change).

Hamming Code

Error correcting code. Parity bits at positions 2⁰, 2¹, 2², … (1, 2, 4, 8, …). Can detect 2-bit errors and correct 1-bit errors. SECDED (Single Error Correct, Double Error Detect) adds an extra parity bit. Used in ECC memory.

For m data bits: need r parity bits where 2^r ≥ m + r + 1

Forward Error Correction (FEC)

Receiver corrects errors without retransmission. Used in lossy links, satellite, storage, 5G. Examples:

  • Reed-Solomon — used in CDs, DVDs, QR codes, deep-space comms
  • LDPC (Low-Density Parity Check) — near Shannon capacity, used in 10GbE, WiFi 6, 5G NR
  • Turbo Codes — used in 3G/4G LTE
  • Polar Codes — 5G control channel
